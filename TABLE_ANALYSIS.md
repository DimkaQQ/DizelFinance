## ❌ НАЙДЕННЫЕ ПРОБЛЕМЫ

### ПРОБЛЕМА 1: Несоответствие категорий между ботом и таблицей

**Что не так:**
В документах указаны категории для бота:
- Крупные покупки
- Консалтинг

Но в листе "Бюджет 2025" могут быть **другие названия строк** или категории могут отсутствовать.

**Как исправить:**

1. Откройте лист "Бюджет 2025"
2. Убедитесь, что есть строки:
   ```
   КРУПНЫЕ ПОКУПКИ
   КОНСАЛТИНГ
   ```
3. Если их нет — добавьте вручную

**Формулы для автоподсчёта:**

```excel
// Для строки "КРУПНЫЕ ПОКУПКИ" (например, строка 30)
// Колонка B (январь):
=SUMIFS(Транзакции!D:D, Транзакции!B:B, "Крупные покупки", Транзакции!A:A, ">=01.01.2025", Транзакции!A:A, "<=31.01.2025")

// Для строки "КОНСАЛТИНГ"
=SUMIFS(Транзакции!D:D, Транзакции!B:B, "Консалтинг", Транзакции!A:A, ">=01.01.2025", Транзакции!A:A, "<=31.01.2025")
```

### ПРОБЛЕМА 2: Формулы не учитывают новые категории

**Что не так:**
В листе "Бюджет 2025" есть строка **"ИТОГО РАСХОДЫ"**, но формула суммирует только **старые категории** (Жизнь, Дом, Транспорт...), а новые (Крупные покупки, Консалтинг) не включены.

**Как исправить:**

Найдите ячейку с формулой типа:
```excel
=SUM(B10:B25)
```

Измените диапазон, чтобы включить новые строки:
```excel
=SUM(B10:B32)  // Если добавили 2 новые категории
```

Или используйте **именованные диапазоны**:
```excel
=SUMIF(B:B, ">0")  // Суммирует все положительные значения в колонке B
```

### ПРОБЛЕМА 3: Дублирование данных между "Консалтинг" и "Бюджет 2025"

**Что не так:**
В документе написано, что в "Бюджет 2025" попадает **ДЕЛЬТА** (доходы минус расходы) от консалтинга, а не валовые доходы.

**Проверка:**
```
Лист "Консалтинг":
Янв: доходы 103,700 - расходы 41,926 = 61,774 ₽

Лист "Бюджет 2025", строка "Консалтинг":
Янв: 61,774 ₽ ✅ (совпадает — это дельта, всё правильно)
```

**Если не совпадает — исправьте формулу:**

В листе "Бюджет 2025", ячейка B20 (например, январь, строка "Консалтинг"):
```excel
='Консалтинг'!B50
```
где `B50` — ячейка с **ДЕЛЬТОЙ** (доходы - расходы) за январь в листе "Консалтинг".

### ПРОБЛЕМА 5: Неправильные диапазоны дат в формулах

**Что не так:**
Формулы используют фиксированные даты типа:
```excel
">=01.01.2025"
```

Это означает, что в феврале нужно будет **вручную менять** все формулы на `">=01.02.2025"`.

**Как исправить:**

Используйте **динамические диапазоны**:

```excel
// Для января (ячейка B10):
=SUMIFS(Транзакции!D:D, Транзакции!B:B, "Жизнь", Транзакции!A:A, ">="&DATE(2025,1,1), Транзакции!A:A, "<="&EOMONTH(DATE(2025,1,1),0))

// Где:
// DATE(2025,1,1) — первое число месяца
// EOMONTH(DATE(2025,1,1),0) — последнее число месяца
```

**Ещё лучше — привязать к заголовку колонки:**

Если в ячейке B1 написано "Январь 2025", то:
```excel
=SUMIFS(Транзакции!D:D, Транзакции!B:B, "Жизнь", Транзакции!A:A, ">="&DATE(YEAR(B$1),MONTH(B$1),1), Транзакции!A:A, "<="&EOMONTH(DATE(YEAR(B$1),MONTH(B$1),1),0))
```

Тогда при копировании формулы вправо (в февраль, март) — даты будут обновляться автоматически.

### ПРОБЛЕМА 7: Формат даты в листе "Транзакции"

**Что не так:**
Бот записывает дату в формате `20.01.2025`, но Google Sheets может интерпретировать это как **текст**, а не как дату.

**Последствия:**
- Формулы с `SUMIFS` и `DATEVALUE` не работают
- Невозможно фильтровать по датам

**Как исправить:**

1. Выделите колонку A (Дата) в листе "Транзакции"
2. Меню: Формат → Число → Дата
3. Выберите формат: `20.01.2025`

**В коде бота — уже исправлено:**
```python
date_record = datetime.now().strftime("%d/%m/%Y")  # Слеш вместо точки
```

Но если хотите точки — убедитесь, что Google Sheets распознаёт формат `dd.mm.yyyy`.

### ПРОБЛЕМА 8: Отсутствие валидации данных

**Что не так:**
В листе "Транзакции" нет ограничений на вводимые данные:
- Можно записать отрицательную сумму
- Можно записать неправильную категорию (опечатка)

**Как исправить:**

#### A. Валидация категорий (колонка B):
1. Выделите колонку B (Категория)
2. Меню: Данные → Проверка данных
3. Критерий: "Значение из диапазона"
4. Диапазон: создайте лист "Справочники" со списком категорий
5. Установите флажок "Отклонять ввод при недопустимых данных"

#### B. Валидация суммы (колонка D):
1. Выделите колонку D (Сумма)
2. Меню: Данные → Проверка данных
3. Критерий: "Больше или равно 0"

### ПРОБЛЕМА 9: Нет защиты от дублирования транзакций

**Что не так:**
Если бот запишет одну и ту же транзакцию дважды (например, из-за ошибки или повторного SMS) — в таблице будет **дубль**.

**Как исправить:**

#### Вариант 1: Условное форматирование (выделять дубли)
1. Выделите все данные в листе "Транзакции"
2. Меню: Формат → Условное форматирование
3. Формула:
   ```excel
   =COUNTIFS($A:$A,$A1,$B:$B,$B1,$D:$D,$D1)>1
   ```
4. Формат: Красный фон (или другой цвет)

#### Вариант 2: Защита на уровне кода (в боте)
В `finance_bot.py` добавить проверку перед записью:
```python
# Перед ws.append_row(new_row)
existing_records = ws.get_all_records()
for rec in existing_records:
    if (rec['Дата'] == data['date'] and 
        rec['Категория'] == data['category'] and 
        rec['Сумма'] == data['amount']):
        await message.answer("⚠️ Такая транзакция уже существует!")
        return
```

## ✅ РЕКОМЕНДАЦИИ ПО УЛУЧШЕНИЮ

### 1. Создайте лист "Справочники"

Создайте отдельный лист со списками:
- Колонка A: Категории
- Колонка B: Подкатегории для "Жизнь"
- Колонка C: Подкатегории для "Дом"
- Колонка D: Подкатегории для "Консалтинг"
- Колонка E: Названия карт/счетов

**Преимущества:**
- Легко добавлять новые категории (без изменения кода)
- Валидация данных через выпадающие списки

### 2. Используйте именованные диапазоны

Вместо `Транзакции!D:D` используйте `Суммы`:
1. Выделите колонку D в листе "Транзакции"
2. Меню: Данные → Именованные диапазоны
3. Имя: `Суммы`

Теперь формулы выглядят так:
```excel
=SUMIFS(Суммы, Категории, "Жизнь", Даты, ">=01.01.2025")
```

### 3. Добавьте сводные таблицы

Создайте лист "Сводная" с Pivot Table:
1. Меню: Вставка → Сводная таблица
2. Диапазон: весь лист "Транзакции"
3. Строки: Категория
4. Значения: Сумма (SUM)
5. Фильтры: Дата (по месяцам)

### 4. Настройте автоматическое резервное копирование

Google Sheets автоматически сохраняет версии, но для надёжности:
1. Установите Google Drive на компьютер
2. Включите синхронизацию
3. Локальная копия будет обновляться автоматически

Или создайте скрипт (Google Apps Script):
```javascript
function backupSheet() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var folder = DriveApp.getFolderById('YOUR_FOLDER_ID');
  var date = Utilities.formatDate(new Date(), 'GMT+6', 'yyyy-MM-dd');
  ss.copy('Backup_' + date);
}
```
